+ PARA 是一種個人知識管理架構，由 Tiago Forte 在其著作《打造第二大腦》中提出。它的目標是幫助人們管理數位資訊。以下是 PARA 的摘要條列：

   + **四大分類**：PARA 將所有內容分為四個主要類別：

      - **P (Projects)**：專案，指有明確目標和截止日期的短期工作。

      - **A (Areas)**：領域，指需要持續關注和維護的長期責任範圍。

      - **R (Resources)**：資源，指未來可能有用，但目前與特定專案或領域無關的資料或興趣點。

      - **A (Archives)**：歸檔，指已經完成或不再活躍的專案、領域或資源。

   + **CODE 工作流程**：PARA 系統透過 CODE 四個步驟進行知識管理：

      - **Capture (獲取)**：收集外部資訊。

      - **Organize (整理)**：將收集到的資訊歸類到 PARA 的四個類別中。

      - **Distill (萃取)**：提煉和總結資訊的核心要點。

      - **Express (表達)**：將萃取出的知識轉化為輸出和產出。

   + **工具通用性**：這套方法不挑筆記工具，無論您使用 Notion、Evernote、Obsidian 或是 Heptabase，都能夠應用。
